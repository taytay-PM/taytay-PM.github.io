过渡效果的实现

本部分过渡效果的实现整体思路是，先在卡片中填入文字并将其布局居中，后用圆形元素将其覆盖，而当鼠标移动到卡片上时圆形元素内的图标会瞬间消失，再衔接上卡片和图片的hover效果，以实现一个绚丽的过渡。具体实现过程如下：

1、卡片hover效果

.card:hover {
  width: 600px;
  transition-delay: 0.5s;
}
  • 当鼠标悬停在卡片上时，卡片的宽度会从初始的310px过渡到600px。transition-delay属性指定了过渡效果延迟0.5秒触发，即在悬停0.5秒后开始过渡。

2、圆形背景过渡

  .card .circle::before{
    content:'';
    position: absolute;
    top:0;
    left:0;
    border-radius:50%;
    width:350px;
    height:350px;
    background: #191919;
    border:8px solid var(--clr);
    filter:drop-shadow(0 0 10px var(--clr)) drop-shadow(0 0 60px var(--clr));
    transition: 0.5s,background0.5s;
    transition-delay:0.75s,1s;

  }
  .card:hover .circle::before {
  width: 100%;
  height: 100%;
  border-radius: 20px;
  background: var(--clr);
  transition-delay: 0.5s;
}

圆形元素的背景过渡效果由::before伪元素控制。当鼠标悬停在卡片上时，::before伪元素的宽度和高度会从350px过渡到100%（相当于圆形元素的宽度和高度）且圆角减低成20px，这样做是为了把card部分覆盖掉。背景色从初始的黑色过渡到定义的颜色--clr（在html中的style里设置）。过渡效果延迟0.75秒开始，背景颜色的过渡延迟1秒开始，这样的先后次序更有层次感

3、圆形内图标过渡

.card:hover .circle .logo {
  transform: scale(0);
  transition-delay: 0s;
}

当鼠标悬停在卡片上时，圆形元素内的图标会从初始的原始大小（1倍）过渡到scale(0)，即消失，过渡效果立即开始。

4、图像的过渡

.card .img{
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%,-50%)scale(0) rotate(315deg);
  height: 270px;
  transition: 0.5s ease-in-out;
}

.card:hover .img{
  transition-delay: 0.75s;
  top:50%;
  left: 72%;
  height: 500px;
  transform: translate(-50%,-50%) scale(1) rotate(8deg);
}

图像在卡片中的位置和大小发生过渡。当鼠标悬停在卡片上时，图像会从初始的scale(0)过渡到scale(1)，即从不可见到可见状态。同时，图像的位置会从初始的垂直和水平居中过渡到垂直居中、水平偏右的位置,这样做为了不挡到文字内容。过渡效果延迟0.75秒开始。

5、内容的过渡

  opacity: 1;

在HTML中，创建了一个包含轮播图的容器，使用id="banner"来标识该容器。轮播图的图片被包裹在具有class="slider"的<div>元素中，并使用<img>标签添加了多个图片，每个图片都有一个src属性指向不同的图片路径，以及一个alt属性用于图像无法加载时的替代文本。

document.addEventListener('DOMContentLoaded', function() {
  showImage(0);
  setInterval(nextImage, 3000);
});

在JavaScript中，通过监听DOMContentLoaded事件来确保DOM已完全加载后执行代码。在事件处理程序中，首先调用showImage(0)来显示第一张图片，然后使用setInterval函数设置一个定时器，每隔3秒调用nextImage函数，实现自动切换图片的效果，这个过程实现的是刚进入页面显示图片的功能。

const slider = document.querySelector('.slider');
const images = slider.getElementsByTagName('img');
let currentIndex = 0;

function showImage(index) {
  for (let i = 0; i < images.length; i++) {
    images[i].classList.remove('active');
  }
  images[index].classList.add('active');
}

function nextImage() {
  currentIndex++;
  if (currentIndex >= images.length) {
    currentIndex = 0;
  }
  showImage(currentIndex);
}

setInterval(nextImage, 3000);

使用querySelector选择器获取具有class="slider"的容器元素，并将其赋值给变量slider。然后，使用slider.getElementsByTagName('img')获取所有图片元素，并将其赋值给变量images。

接着定义两个函数：showImage和nextImage。showImage函数用于显示指定索引的图片，遍历所有图片元素，并为当前索引的图片添加active类，同时移除其他图片的active类，从而控制显示当前图片。nextImage函数用于切换到下一张图片，它通过递增currentIndex变量实现，如果currentIndex超出了图片数组的长度，将其重置为0，然后调用showImage函数显示对应索引的图片。

最后，使用setInterval函数每隔3秒调用一次nextImage函数，实现自动切换图片的效果。

通过上述代码，实现了一个简单的轮播图功能。

思路在于利用滚动事件和元素的样式修改来实现动态效果。通过适当调整样式属性的变化速率和方向，实现不同元素在滚动过程中以不同速度和方向移动，从而创造出更加生动的滚动效果。

通过document.getElementById方法获取了具有相应id的元素，并将它们分别赋值给变量text、leaf、hill1、hill4和hill5

在窗口上添加了一个scroll事件监听器。每当用户滚动页面时，触发该事件处理函数。在处理函数中，首先通过window.scrollY·获取当前滚动的垂直距离，并将其赋值给变量value。

然后，通过修改各元素的样式，根据value的值来实现滚动效果。具体来说：

text.style.marginTop = value * 1.5 + ‘px’;：根据滚动距离调整text元素的顶部边距，以实现向下移动的效果。
leaf.style.top = value * -1.5 + ‘px’;：根据滚动距离调整leaf元素的顶部位置，以实现向上移动的效果。
leaf.style.left = value * 1.5 + ‘px’;：根据滚动距离调整leaf元素的左侧位置，以实现向右移动的效果。
hill5.style.left = value * 1.5 + ‘px’;：根据滚动距离调整hill5元素的左侧位置，以实现向右移动的效果。
hill4.style.left = value * -1.5 + ‘px’;：根据滚动距离调整hill4元素的左侧位置，以实现向左移动的效果。
hill1.style.top = value * 0.5 + ‘px’;：根据滚动距离调整hill1元素的顶部位置，以实现向下移动的效果。
通过改变元素的样式属性，随着滚动事件的触发，元素将相应地移动，从而产生视差效果。

使用 CSS 伪类选择器 :nth-of-type()，用于选择同类型的元素中的特定位置的元素。使用 :nth-of-type() 来快速选择列表中的不同位置的列表项（<li> 元素）。

通过简单的js逻辑可以实现轮播图的效果，首先，通过document.querySelector('.slider')获取到页面中具有slider类名的元素，然后通过slider.getElementsByTagName('img')获取到该元素下的所有img标签。然后，定义了一个变量currentIndex来表示当前显示的图片的索引，默认为0，在设置定时器后，遍历索引，在索引超过图片数量时，将currentIndex重置为0。

但是会出现进入页面后没有显示任何图片，需要等一会才能显示出第一张图片，因此我在js代码前面补充了一个事件的监听器

这段代码使用了DOMContentLoaded事件监听器来确保在页面加载完毕后执行代码。在页面加载完毕后，调用showImage(0)函数来显示第一张图片，并使用setInterval函数每隔3秒钟调用nextImage函数，实现自动切换图片的效果。

3.5 跳转页面影响对齐问题

该问题是在做项目的过程中前期的遇到的问题，前期我使用到是侧边栏进行与用户交互，然后再侧边栏的右边区域显示跳转的页面，这时就遇到了跳转后页面空白和跳转后侧边栏与主内容区不对齐的问题。

关于页面空白的解决方案是：
根据您的描述，将侧边栏页面的body样式中的display: flex;、justify-content: center;和align-items: center;删除后，跳转到的.html可以在主内容区正常展示。

这是因为display: flex;与justify-content: center;和align-items: center;属性组合在一起，会使body元素成为一个弹性容器，并将其内部的子元素在水平和垂直方向上居中显示。这可能导致body元素占据整个可视窗口的高度，从而覆盖主内容区域。

通过删除这些样式，body元素将恢复其默认的块级元素行为，并且不会影响其他元素的布局。因此，.html可以在主内容区正常展示，不再被body元素的样式所影响。

关于页面不对齐的问题：

.home-section {
      background: var(--home-bgCololr);
      display: flex;
    flex-direction: column;
    height: auto; /*解决了跳转后页面不对齐问题 */
    }

我将主内容区的高度设置成auto，即解决了该问题。

5 项目总结

本次项目我做了一个关于“我喜欢的”为主题的静态网站项目，在制作的过程中，从开始的分析阶段到实现过程中的不断的试错阶段都提升了我的前端设计与编码能力，熟悉了相关属性的作用以及样式的含义，再加上JavaScript的逻辑部分的实现，可以巧妙地制作出我想要的效果（比如专辑页面的动画效果，看似复杂实则是两个不同形状卡片的切换过渡，与图片的样式透明度设置）。利用切换时间的设置不同以实现最终的效果。还有在想到要实现一个图片移动的效果时，可以换个角度考虑，以图片跟父元素之间的距离为出发点，调整他们的距离，移动的方向就是相反的，将这个思路写到JavaScript里面就实现了图片的移动效果。而“电影模块”的轮播图实现，则通过查阅相关资料了解一些比较系统的解决方案后，再进行的一个设计，虽然存在一些问题比如轮播图中的图片有时会拉伸严重，造成美观上的不足，因此我只能调高其高度来实现拉伸的减缓，但这样就会显得轮播图很巨大。

但还有很多不足之处，比如页面的美化、文字部分的美化、布局还可以优化以及页面切换时的平滑效果，轮播图没有设计箭头的效果，用户无法自己切换轮播图。这些方面和市面上的网页设计还是存在一定的差距，因此还需要继续努力的改进。

而针对这些相关的问题，我也想到一些解决方案与展望：

  • 关于页面美化：考虑使用合适的配色方案（ColorDrop网站，虽然这次有用到但是用的不够）和字体组合来提升整体的美观度。参考一些优秀的网站设计以及色彩理论来选择合适的颜色搭配。此外，使用吸引人的高质量图片作为背景或插图也可以提升网站的外观。

  • 关于文字美化：选择适合网站风格的字体，并为标题、段落和链接等元素应用合适的样式，如字体大小、行高和字间距。可以使用CSS的文本属性来实现这些效果。

  • 关于布局优化：本次弹性布局使用的比较多，但使用网格系统或弹性盒子布局也可以更好地控制页面的布局。应该熟练掌握这两种布局方式。

  • 平滑过渡效果：通过使用CSS过渡或动画来实现页面切换时的平滑过渡效果。这些过渡效果可以为用户提供更好的视觉体验，并增强网站的专业感。

  • 轮播图改进：考虑添加箭头按钮或指示器来让用户自己切换轮播图。这样用户可以根据自己的喜好自由浏览图片。同时，为了确保轮播图中的图片不会拉伸变形，以后可以尝试使用适当的图片大小或裁剪图片来适应轮播图的尺寸。

所以继续努力改进你的网站是非常重要，不断学习和尝试新的设计和编码技术，关注用户的反馈，并寻求灵感和参考于其他优秀的网站设计，来不断提升你的网站的质量和用户体验。
